La importancia de los índices en las búsquedas de bases de datos

Para llevar a cabo la búsqueda en nuestra base de datos, necesitamos realizar tres pasos esenciales. Primero, debemos definir el índice en el que realizaremos la búsqueda. Segundo, seleccionar el rango de búsqueda adecuado. Y tercero, abrir un cursor y proporcionarle el rango como parámetro. No te preocupes si estas palabras clave te suenan un poco confusas en este momento; las exploraremos una a una. Empezaremos por definir el índice en el que se realizará la búsqueda.

Recuerda que en el primer capítulo ya habíamos establecido el índice BuscarNombre, ya que sabíamos que realizaríamos búsquedas por el nombre del contacto. Para definir el índice que utilizaremos, empleamos el método index(). Dentro de los paréntesis, especificamos el nombre del índice entre comillas, como se muestra aquí: var indice = almacen.index('BuscarNombre'). Concepto y utilidad de establecer RANGOS de búsqueda

Para definir el rango de búsqueda, contamos con el objeto IDBkeyRange, que ofrece métodos como only(), bound(), lowerBound() y upperBound(). Ahora, independientemente del método que elijamos para definir el rango, debemos proporcionar la variable buscar (en este caso) como parámetro, ya que contiene la búsqueda realizada por el usuario. La forma más sencilla de entender qué es un rango es describir las funciones de estos métodos.

Por ejemplo, si usamos el método only(buscar), obtendremos una lista de nombres que coincidan exactamente con lo que se ingresó en la barra de búsqueda. Por otro lado, si empleamos el método lowerBound(buscar, false), obtendremos una lista que comienza con los resultados que coinciden con la búsqueda, seguidos de otros resultados que pueden o no coincidir con la búsqueda, sin importar su orden.

No profundizaré más en la explicación de cada método por dos razones. En primer lugar, probar cada uno de ellos llevaría mucho tiempo, y podríamos dedicar medio blog solo a este tema. En segundo lugar, el método que tiene más sentido en este caso es only(), ya que nos devuelve resultados idénticos a la búsqueda. No obstante, te proporcionaré un enlace a la documentación oficial donde puedes explorar cómo funcionan los otros métodos: Enlace a la documentación . Para empezar, declararemos la variable rango (var rango =), luego mencionaremos el objeto IDBkeyRange, elegiremos el método a utilizar que en este caso es only() y por último le pasamos la variable 'buscar'. Ajustes y optimizaciones comunes al trabajar con IndexedDB

Si has implementado todos estos cambios que hemos estado haciendo, seguramente notarás que la búsqueda funciona correctamente. No obstante, hay dos ajustes que debemos hacer para mejorar aún más la experiencia del usuario. En primer lugar, queremos que cada vez que el usuario realice una búsqueda, el campo de búsqueda quede vacío y listo para la siguiente consulta. Esto es fácil de lograr, como ya vimos en la entrada anterior sobre cómo almacenar objetos en la base de datos. Simplemente ubicamos el campo que queremos limpiar y le asignamos un valor de comillas vacías de esta manera: document.querySelector('#buscar-nombre').value = ' '.

En segundo lugar, queremos que cada vez que se realice una nueva búsqueda, los resultados anteriores se eliminen para evitar la acumulación de resultados. Esto también es sencillo y lo implementaremos en la función buscarContacto(). En esta función, ubicamos el contenedor que deseamos limpiar y le asignamos un valor vacío a la propiedad innerHTML, como se muestra aquí: document.querySelector('.resultado-busqueda').innerHTML = ' '.

Razones para almacenar y buscar datos en minúsculas

Con todo lo que hemos hecho hasta ahora, podríamos considerar terminado nuestro blog de hoy. Sin embargo, me gustaría compartir un consejo adicional contigo. Si realizas un experimento y guardas un contacto con el nombre en mayúsculas y otro con el mismo nombre en minúsculas, al buscar uno de los dos, te llevarás una sorpresa. A pesar de que es el mismo nombre, el resultado de la búsqueda dependerá de cómo se haya almacenado y cómo se esté buscando. Para garantizar un funcionamiento coherente, es esencial tener un control total sobre cómo se almacena el nombre y cómo se realiza la búsqueda.

No te preocupes, esto es más sencillo de lo que parece. Solo necesitas convertir a minúsculas tanto el nombre que se guarda en la base de datos como el que se utiliza en la búsqueda. Permíteme mostrarte un ejemplo con el nombre del contacto para que puedas ver cómo hacerlo. A continuación, te proporciono la línea de código que debes agregar en la función AlmacenarContacto() para transformar el nombre al guardarlo en minúsculas: document.querySelector('#nombre').value.toLowerCase().

De manera similar, puedes realizar este cambio en la función buscarContacto() para convertir el término de búsqueda a minúsculas. Esto garantizará que las búsquedas sean insensibles a las mayúsculas y minúsculas, lo que mejorará significativamente la experiencia del usuario. Mejorando la presentación de texto con la propiedad text-transform

Ahora, con estos cambios, tenemos un control total sobre cómo se almacena el nombre del contacto y cómo se realiza la búsqueda. Sin embargo, es posible que notes que tanto la lista de contactos como los resultados de la búsqueda se muestran en minúsculas, lo cual no luce muy ordenado. Afortunadamente, también podemos controlar cómo queremos que se presenten ambos conjuntos de resultados. Para lograrlo, simplemente necesitamos dirigirnos al archivo CSS y, en la sección de estilos para estos dos contenedores, agregamos la propiedad text-transform con el valor capitalize (text-transform: capitalize). Esto formateará correctamente los nombres para que la primera letra de cada palabra esté en mayúscula, independientemente de cómo se hayan guardado en la base de datos.
