CSS Grid desde Cero: Diseños Adaptables Sin Necesidad de Media Queries.

¿Es posible crear un layout responsive sin recurrir a media queries? ¡Por supuesto que sí, Código-Lover! Y para lograrlo, entra en escena CSS Grid. En este blog, vamos a partir desde cero para construir un layout utilizando CSS Grid y llevarlo hasta su versión responsive. No te preocupes si esta es tu primera incursión en el tema, voy a guiarte paso a paso para que no te quedes con ninguna pregunta sin respuesta. Lo primero que necesitas hacer para comenzar a establecer tu sistema grid es declarar un contenedor (caja padre) junto con las cajas hijos en tu hoja de HTML. En esta ocasión, solo utilizaremos ocho cajas hijo. Asignaré un id al contenedor y dos clases a las cajas hijo. La primera clase se llamará 'entrada', que será igual para todas. La segunda clase se nombrará 'ent_' seguido del número correspondiente a cada una. Esto nos permitirá trabajar con cajas específicas si llegamos a necesitarlo.

Ahora que hemos definido las cajas en HTML, es momento de dirigirnos a CSS para darles estilo. El contenedor padre tendrá un ancho del 90% (width), un margen de 40px tanto arriba como abajo, márgenes automáticos a los lados para mantenerlo centrado, un padding, una sombra y un fondo blanco. En cuanto a las cajas hijo, simplemente les añadiré un fondo blanco, texto centrado, bordes redondeados (border-radius) y padding. Estos son estilos que se utilizan comúnmente, así que no me detendré a explicarlos uno por uno. A partir de este código que te proporcionaré a continuación, comenzaremos a dar forma al layout con CSS Grid. También adjuntaré una imagen para que puedas echar un vistazo, incluso si no estás frente a tu ordenador.

Primeros Pasos con CSS Grid: Construyendo tu Layout

Antes de adentrarnos en el fascinante mundo de los layouts con CSS Grid, quiero aclarar un detalle importante. En el código que vas a ver, he utilizado variables porque me encanta trabajar con ellas. Si aún no estás familiarizado con el concepto de Variables en CSS, te invito a echar un vistazo a este blog: 'Variables en CSS: ¿Cómo aprovecharlas al máximo?' Te prometo que comprender su funcionamiento es realmente sencillo, ¡vale la pena descubrirlo!

Ahora sí, vamos de lleno a explorar el emocionante mundo del layout con CSS Grid. Lo primero que debes hacer es declarar el contenedor, también conocido como la caja padre. Para ello, utiliza la propiedad 'display' con el valor 'grid'. A partir de este punto, comenzamos a construir la estructura de nuestra retícula. Ten presente que CSS Grid se basa en filas y columnas. Para diseñar tu retícula, define cuántas columnas y filas deseas en tu layout. Por ejemplo, en este caso he optado por 4 columnas y 2 filas. Para establecer las columnas, emplea la propiedad 'grid-template-columns'; mientras que para las filas, usa 'grid-template-rows'. Dentro de estas propiedades, declara las medidas de cada columna o fila que necesites. Si, por ejemplo, deseas que haya 4 columnas con un ancho del 25% cada una, la declaración sería así: 'grid-template-columns: 25% 25% 25% 25%'. El proceso es idéntico para las filas. Dado que tenemos 8 cajas en total, solo necesitamos definir dos filas, lo que quedaría así: 'grid-template-rows: 200px 200px'. Además, puedes combinar estas dos propiedades en una sola utilizando 'grid-template'. Al hacerlo, primero especifica las filas necesarias y luego, separadas por una barra diagonal, las columnas deseadas. Si copias y pegas los valores de las propiedades anteriores, obtendrás exactamente el mismo resultado en tu diseño.

¿Cómo Funciona 'gap'? Distancias y Margenes en CSS Grid.

Con las filas y columnas definidas, es hora de darles un poco de espacio. Por defecto, estas se encuentran completamente ajustadas, pero vamos a solucionarlo utilizando la propiedad 'gap'. (Cabe mencionar que anteriormente se empleaba 'grid-gap', pero ha quedado en desuso. Si deseas profundizar en este tema, te recomiendo este tutorial: 'CSS Grid: ¿Cómo declarar filas y columnas?') Esta propiedad admite dos valores: el primero determina el margen horizontal y el segundo el margen vertical. Si proporcionas solo un valor, este será aplicado en ambos lados. Personalmente, he decidido asignar un margen de 20px. Ahora, es importante detenernos aquí para que puedas comprender qué ocurre cuando aplicamos este margen. Verás que surge un pequeño inconveniente en nuestro diseño. En el código que te proporcionaré más adelante, encontrarás ejemplos de cómo declarar filas y columnas, lo que te ayudará a entender mejor la sintaxis y la dinámica de este proceso.

Primero, quiero señalarte que en el código te proporcioné un comentario para que puedas comprender de manera clara los dos ejemplos. Si así lo deseas, puedes eliminar sin preocupación las dos primeras líneas, y notarás que la estructura se mantiene inalterada. Ahora bien, como has observado en la imagen o en tu propia pestaña del navegador si copiaste el código, las cajas hijas sobresalen del contenedor padre. Este fenómeno ocurrió después de agregar el "grid-gap". ¿A qué se debe este comportamiento? Resulta que, aunque estemos familiarizados con el uso de porcentajes para que la estructura se ajuste a distintos tamaños de ventana, en CSS Grid las cosas funcionan de manera diferente. En realidad, el "grid-gap" de 20px se suma al 25% de cada caja. Esto implica que al final superamos el 100% de ancho total, lo que resulta en que la última caja se desborde del contenedor.

La Revolucionaria Unidad "fr": Entendiendo las Fracciones en CSS Grid

Para trabajar con CSS Grid, contamos con una unidad que presenta similitudes con la propiedad "flex" en las cajas flexibles. Esta unidad se denomina "fr" (fracción). En lugar de especificar que cada columna ocupe un 25%, emplearemos "1fr". En términos más simples, esto implica que cada columna ocupará una fracción del espacio disponible. En este contexto, obtendremos el mismo resultado que cuando usamos porcentajes, pero evitando el problema que enfrentamos anteriormente. Si te surge alguna pregunta respecto a cómo se declara, puedes revisar el código que te proporciono a continuación y luego proseguir con la lectura.

¿Repitiendo Columnas y Filas? Conoce la Función "repeat"

¡Bien! Ahora que hemos establecido las filas y las columnas, y resuelto el problema, quiero presentarte una forma más concisa para declararlas. En lugar de agregar columnas una por una, como hicimos en ejemplos anteriores, podemos emplear la función "repeat()". Esta función acepta dos parámetros: el primero indica cuántas veces deseamos repetir y el segundo, separado por una coma, establece el valor a repetir. Para las columnas, podríamos utilizar "repeat(4, 1fr)", mientras que para las filas, sería "repeat(2, 200px)". Además, es relevante saber que el segundo parámetro, es decir, el valor a repetir, puede contener varios valores. Por ejemplo, podríamos indicar que se repita dos veces una columna de 200px y luego otra de 1fr, obteniendo un total de cuatro columnas: dos de cada tipo. A continuación, encontrarás ejemplos que ilustran todas estas posibilidades.

Maestría en Posicionamiento: Alineando Elementos con CSS Grid

Ahora que hemos establecido la estructura básica, es momento de adentrarnos en el posicionamiento de las cajas. En este ejemplo, vamos a tratar con cajas de igual tamaño. Sin embargo, antes de sumergirnos en el posicionamiento en sí, hay un concepto crucial que necesitas comprender. Te lo ilustro en la siguiente imagen. Para lograr un posicionamiento preciso de las cajas, es fundamental entender cómo funcionan las líneas en la cuadrícula. En la imagen, observarás que cada línea que divide las cajas está numerada. Si nos enfocamos en las columnas, notaremos que comienzan con el número 1 desde la izquierda y avanzan hasta la última línea, marcada como 5. Lo mismo sucede con las filas: en la parte superior e izquierda, encontramos la línea 1, mientras que en la esquina inferior izquierda, aparece la línea 3. Así es también desde la perspectiva opuesta, aunque en este caso los números son negativos. No te preocupes por los valores negativos, te explicaré su uso más adelante para que todo quede claro.

"grid-column": Una Herramienta Esencial para Manejar Columnas

Ahora que tenemos en mente los números que representan las líneas, podemos empezar a desplazar la "entrada 1" hacia la posición de la "entrada 2". Para lograr esto, utilizaremos la propiedad "grid-column-start". Para modificar específicamente la posición de la "entrada 1", utilizaremos la segunda clase que habíamos definido anteriormente, es decir, "ent_1". En esta clase, añadiremos la propiedad mencionada y asignaremos el número de la línea desde donde deseamos que inicie, en este caso, el número 2. La declaración se vería así: "grid-column-start: 2;". Con este ajuste, notarás cómo la "entrada 1" se desplaza a la posición de la "entrada 2". Sin embargo, esto es solo el comienzo.

También tenemos la posibilidad de indicar que la caja ocupe más de un espacio. Para lograrlo, debemos utilizar dos propiedades: la primera, que ya te he mostrado, determina dónde comienza la caja; la segunda es "grid-column-end", que especifica dónde termina. En este caso, para que la "entrada 1" ocupe dos espacios, debería finalizar en la línea 4. Así que el valor que emplearemos en la segunda propiedad será "grid-column-end: 4;". Es importante saber que estas dos propiedades pueden abreviarse en una sola, utilizando "grid-column". El primer valor indicará el inicio, el segundo el final y se separan por una barra diagonal. Puedes observar este código en el comentario que he dejado en el código fuente. Si lo deseas, échale un vistazo y continúa con la lectura.

"grid-row": Diseño Vertical en CSS Grid.

Hasta ahora, hemos logrado mover la caja de posición y hacer que ocupe más espacio en las columnas. Ahora, vamos a expandirla verticalmente, es decir, que abarque más de una fila. Aunque las propiedades a utilizar son diferentes, la sintaxis es exactamente la misma. Para definir dónde comienza nuestra caja en términos de filas, usaremos la propiedad "grid-row-start". Por otro lado, para indicar dónde termina, recurrimos a "grid-row-end". Estas dos propiedades pueden ser combinadas en "grid-row", que sigue la misma estructura que las propiedades de columnas. Es importante señalar que la sintaxis es idéntica a las propiedades de columnas, aunque con el enfoque en filas. Todos estos conceptos están ilustrados en los ejemplos de código que te proporciono a continuación, separados por comentarios para que no quede ninguna duda. Primero, encontrarás los ejemplos y las imágenes; después, resolveremos un pequeño problema que notarás en la siguiente imagen.

Creando Filas Dinámicas con "grid-auto-rows"

Ahora, quiero aprovechar la oportunidad para abordar el tema de los números negativos en las líneas, ya que te he explicado cómo posicionar elementos utilizando números positivos. De hecho, cuando trabajamos con números positivos o negativos, en última instancia, la metodología es la misma. Simplemente es crucial estar atentos a qué número corresponde a la línea de inicio de tu elemento y cuál se asigna a su línea final. Más allá de esta distinción, no existe ninguna diferencia en su aplicación.

Hasta ahora, hemos logrado cambiar la posición de una de las cajas e incluso ampliar su ocupación a lo largo de 4 espacios. Sin embargo, al hacer que una caja abarque 4 espacios, las demás cajas se ven obligadas a ajustarse, lo que a menudo puede resultar en la creación de una nueva fila. Este es un comportamiento predeterminado de CSS Grid. Sin embargo, tenemos la capacidad de modificar este comportamiento mediante la propiedad "grid-auto-flow". Por defecto, su valor es "row" (filas). Aunque técnicamente puedes cambiarlo para que en lugar de crear filas, se generen columnas de manera automática, personalmente, encuentro que esta opción no es muy útil. Después de todo, tenemos un límite de ancho en la pantalla, mientras que en la dirección vertical, prácticamente no hay restricciones de espacio.

Adicionalmente, contamos con la propiedad "grid-auto-rows" que nos permite ajustar la altura de estas filas automáticas. Dado que hemos asignado una altura de 200px a las filas, ese mismo valor se utilizará en esta nueva propiedad. Esto resuelve eficazmente el problema que habíamos enfrentado. Vale la pena destacar que "grid-auto-rows" debe aplicarse en el contenedor padre, ya que es este el que define la estructura de la retícula.

El Arte del Posicionamiento: Cómo Utilizar el Comando "span"

Permíteme retroceder un paso y retomar el tema del posicionamiento. En el contexto de CSS Grid, una herramienta valiosa es la palabra clave "span", la cual hace referencia a la extensión de espacio. Permíteme explicarte cómo funciona: en nuestra retícula actual, hemos posicionado la "entrada 1" para que se inicie en la línea 2 y concluya en la línea 4 en términos de columnas. En lugar de hacerlo de esta manera, podríamos utilizar la palabra "span" para lograr el mismo resultado. Es decir, podríamos decirle a la "entrada 1" que comience en la línea 2 y que abarque dos "span" (span 2). Como puedes ver, dentro de los paréntesis, escribimos primero la palabra "span" y luego el número de espacios que deseamos ocupar. Alternativamente, podríamos indicar dónde queremos que termine, como en la línea 4, y que ocupe dos espacios previos a dicha línea. A continuación, en el código que te proporciono, encontrarás ejemplos de ambos casos. Uno define dónde inicia en términos de columnas y luego ocupa dos "span". El segundo ejemplo hace referencia a las filas; aquí, primero le decimos que ocupe dos "span" y luego que finalice en la línea 3. Te invito a revisar el código y la imagen para comprender las propiedades finales que harán que este diseño sea completamente responsive sin la necesidad de utilizar Media Queries.

Máxima Flexibilidad con Auto-Fill: La Magia de las Dimensiones en CSS Grid

En este ejemplo, he optado por declarar todas las cajas con dimensiones idénticas. Las filas mantendrán una altura de 200px, mientras que las columnas adoptarán un enfoque diferente. Utilizaré nuevamente la función "repeat()" para definir cuatro columnas, pero en esta ocasión, en lugar de asignar una fracción (1fr) a cada una, estableceré un ancho fijo de 300px. Como podrás intuir, al llevar a cabo esta acción, la estructura adquirirá una rigidez absoluta. Al modificar el tamaño de la ventana del navegador, notarás que el contenedor principal se contrae o expande, pero las cajas hijas conservan sus dimensiones inmutables.

No obstante, cuando nos enfocamos en desarrollar una estructura responsive, la situación cambia radicalmente, ya que nuestro objetivo es permitir que las cajas se ajusten automáticamente y, en algunos casos, se apilen una encima de otra. Para lograrlo, reemplazaremos el primer parámetro de la función "repeat()" con el valor "auto-fill". Tan solo al implementar este ajuste, observarás cómo las cajas adoptan un comportamiento similar al de los elementos flotantes cuando les indicamos que floten hacia la izquierda. Sin embargo, a pesar de este cambio, las dimensiones de las cajas permanecen invariables, manteniéndose en 300px.

Si deseamos flexibilidad en las dimensiones, será necesario un enfoque diferente, el cual abordaremos en las siguientes secciones para lograr un diseño verdaderamente adaptable.

Minmax(): Dominando las Dimensiones Dinámicas en Grid

Finalmente, para lograr la autorredimensión de estas cajas, solo falta un pequeño ajuste. Es el momento de introducir una nueva función: "minmax()". Esta función reemplazará el segundo parámetro de la función "repeat()". Si alguna duda persiste, no dudes en revisar el código que se encuentra a continuación para obtener más claridad. La función "minmax()", como su nombre sugiere, implica "mínimo" y "máximo". Por ende, el primer parámetro establece el ancho mínimo que nuestras cajas mantendrán, mientras que el segundo parámetro dicta el ancho máximo. Para este caso, especificaré un ancho mínimo de 300px y un ancho máximo de 1fr. Con esta configuración, permitiremos que las cajas crezcan hasta ocupar todo el espacio disponible en función de las limitaciones que establecimos.
