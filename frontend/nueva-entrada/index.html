Configurando Contenedores y Clases para Ocultar y Mostrar Comentarios

He ideado este ejemplo considerando una situación en la que tengamos una sección de testimonios en nuestro sitio web, algunos de los cuales son extensos y necesitamos ocultar parte de ellos. En este escenario, he creado un contenedor principal con el identificador "caja_padre", y tres elementos hijos con la clase "comentario". Cada uno de estos elementos contiene dos párrafos: uno visible en todo momento y otro con la clase "no_mostrar", que permanecerá oculto hasta que el usuario haga clic en el botón "Ver Más" correspondiente al comentario. A continuación, te proporciono el código HTML y CSS, así como una imagen que ilustra cómo se debería ver en el navegador al ejecutar el código.

Si examinas el código CSS, notarás que al párrafo con la clase "no_mostrar" le he asignado la propiedad "display: none" para ocultarlo. Además, he aplicado algunos estilos a una clase llamada "mostrar_comentario", aunque aún no la hemos utilizado. En realidad, solo le he asignado la propiedad "display: block". El objetivo es lograr que, cuando un usuario haga clic en el botón "Ver Más", el contenido adicional del comentario se muestre. Para ello, necesitamos agregar la clase "mostrar_comentario" a la etiqueta del párrafo que tiene la clase "no_mostrar", y esto debe ocurrir en el momento preciso en que el navegador detecta el evento de clic en el botón "Ver Más".

Con esta comprensión clara de lo que deseamos lograr, dirijámonos a nuestra hoja de JavaScript y comencemos. El primer paso es crear dos variables: una para capturar los elementos con la clase "boton" y otra para capturar los elementos con la clase "no_mostrar". La variable para los botones se definiría así (var boton = document.querySelectorAll(".boton")), y para los elementos a mostrar, así (var mostrarComentario = document.querySelectorAll(".no_mostrar")).

Cómo Utilizar NodeList para Manejar Varios Elementos en JavaScript

¿Por qué hemos optado por utilizar querySelectorAll? La razón es que deseamos que la variable contenga todas las clases que comparten el mismo nombre, creando así una lista de nodos. Si colocas un console.log y escribes el nombre de esta variable, verás cómo en la consola se despliega automáticamente un NodeList (lista de nodos) y al lado de cada nodo aparece su posición en la lista. La posición es crucial, ya que todas las clases son idénticas y, sin ella, JavaScript no podría distinguir en qué elemento el usuario ha hecho clic.

Ahora, nuestro objetivo es recorrer esa lista de nodos (NodeList), y esto se logra con el método forEach(). Este método se encarga de ejecutar una acción para cada uno de los elementos contenidos en la lista de nodos (también podemos usarlo en arrays). Para recorrer la lista, simplemente escribimos el nombre de la variable de la lista seguido de un punto y el método forEach(). A medida que avanzamos en el código, te invito a echar un vistazo continuo para observar la sintaxis y aclarar cualquier duda. Dentro de este método, crearemos una función anónima que se encargará de recoger el elemento y su posición en la lista. Hasta este punto, el código luciría así (boton.forEach(function(elemento, indice)){}). El nombre que le demos a los parámetros de la función para recoger el elemento y el índice puede ser similar al que usaríamos al crear una clase, y funcionará sin problemas. Básicamente, con este código, le indicamos a JavaScript que, por cada elemento en la lista de nodos, ejecute lo que le diremos dentro de las llaves de la función. Creo que será beneficioso detenernos aquí por ahora para que asimiles mejor el contenido. En la parte inferior, encontrarás el código hasta este punto.

Añadiendo Event Listeners para Manejar Clics del Usuario

En primer lugar, quisiera recordarte que tanto el código HTML como el CSS permanecen inalterados. Una vez que hemos incorporado el método forEach(), es el momento de crear la función. Dentro de las llaves de esta función, agregaremos un Event Listener al botón. Sin embargo, no podemos asignarlo directamente a la variable "boton", ya que JavaScript no sería capaz de identificar en cuál de los botones estamos haciendo clic. Por tanto, el Event Listener se agregará al elemento específico, de la siguiente manera: (elemento.addEventListener("click", function(){})). Ahora, en el interior de esa función particular, necesitamos que se añada la clase "mostrar_comentario" a la etiqueta de párrafo que ya contiene la clase "no_mostrar" desde el principio.

Recuerda que ya habíamos almacenado otra lista de nodos en la variable "mostrarComentario". Sin embargo, si le agregáramos directamente la clase, nos encontraríamos con el mismo inconveniente que explicaba anteriormente: JavaScript no sabría en qué elemento ejecutar la función. Para garantizar que la clase se añada al elemento correcto, vamos a utilizar el índice de la variable para que JavaScript pueda identificar la posición precisa. Esto se logra así: (mostrarComentario[indice].classList.toggle ("mostrar_comentario")).

Optimiza la Experiencia de Usuario con Instrucciones 'if-else' en JavaScript

Si has copiado el código en tu editor y estás probando su funcionalidad, es posible que hayas notado que aún podemos realizar una corrección para que el comportamiento sea aún más coherente. Me refiero al contenido del propio botón. Como podrás observar, a pesar de que ya se muestre todo el comentario, el botón sigue diciendo "Ver Más". Sin embargo, con una estructura condicional en JavaScript, podemos modificar el contenido del botón para que, cuando la caja esté completamente desplegada, se lea "Ver Menos".

Para establecer esta condición, emplearemos las instrucciones "if" y "else". Al utilizar solo "if", estaríamos abarcando únicamente uno de los estados posibles de dicha condición, específicamente el estado verdadero. Por ejemplo: if (mostrarComentario[indice].classList.contains ("mostrar_comentario")) { elemento.innerHTML = "Ver Menos";}

Como ya mencioné, mediante "if" solo estamos contemplando el estado verdadero de la condición. Esto significa que si la caja se expande automáticamente, el contenido del botón se cambiará a "Ver Menos". Pero, ¿qué sucede cuando la caja vuelve a su tamaño original? Aquí radica la importancia de abordar también el estado falso de la condición. Para resolver este aspecto, simplemente necesitamos añadir "else", seguido de la última llave de "if". La estructura completa se vería así: else {elemento.innerHTML = "Ver Más";} A continuación, proporciono el código completo para que lo examines y puedas apreciar cómo funciona en su totalidad.
