Construyendo un Slider con JavaScript desde Cero

Existen varias formas de hacerlo, y hoy te voy a presentar una de ellas. De hecho, esta será la primera entrada sobre Sliders, ya que las siguientes dos también tratarán este tema. La razón de esto es que hay diferentes enfoques y técnicas para explorar, y creo que es muy interesante compartir varias de ellas, no solo para diversificar el contenido del blog, sino también para ofrecer oportunidades de aprendizaje variadas.

En este primer ejemplo, vamos a construir un Slider que cambiará las imágenes automáticamente cada dos segundos. Además, incluiremos botones para avanzar a la siguiente imagen o retroceder a la anterior, en caso de que el usuario desee un mayor control. Una característica interesante de este ejemplo es que las imágenes no se cargarán desde el HTML, sino directamente desde JavaScript. Pero, no te preocupes, iremos explicando todo esto paso a paso. A continuación, te proporciono el código base para que lo copies y lo desarrolles conmigo.

Empezando con el Código Base

En primer lugar, quiero recordarte que las imágenes que aparecen cargadas en todo el código, ya sea en el HTML, algún fondo en el CSS o las que iremos cargando mientras se desarrolle el JavaScript, no se copiarán automáticamente al copiar el código. Debes cargar tus propias imágenes siguiendo las indicaciones en la sección de Preguntas Frecuentes en caso de dudas. Ahora, retomando el tema principal, como puedes ver, se proporciona tanto el código HTML como el CSS, ya que el JavaScript, como mencioné anteriormente, lo desarrollaremos paso a paso. Antes de sumergirnos en la acción, quiero explicarte lo esencial en el HTML y el CSS para asegurarnos de que no te quedes con ninguna duda.

Desglosando el Código Base: HTML y CSS

En cuanto al HTML, gran parte del maquetado se utiliza para darle un aspecto más atractivo al diseño. Para este modelo de slider, la parte esencial se encuentra entre las dos líneas de comentarios que podrás ver al hacer scroll. En primer lugar, tenemos el contenedor del slider con la clase "contenedor-slider". Dentro de este contenedor, hay dos elementos adicionales, uno al principio y otro al final, que contienen los íconos para desplazar el slider hacia la derecha y hacia la izquierda. Entre estos dos contenedores se encuentra el espacio donde se mostrarán las imágenes. Dentro de ese espacio, tenemos un elemento IMG con el atributo "name" establecido como "Imagen", aunque no carga ninguna imagen en este momento. Explicaré por qué más adelante cuando desarrollemos la parte de JavaScript.

Pasando al CSS, reitero que lo único que nos interesa son los estilos que se encuentran al final, después de la línea de comentarios. Estos estilos son esenciales para que el slider se vea y comporte como se muestra en la imagen que te proporcionaré a continuación. No entraré en detalles de los estilos en este momento para que podamos avanzar rápidamente hacia la parte interesante, que es el código en sí. Dando Vida a tu Slider: Cargando Imágenes con JS

Una vez que hemos explicado el código inicial, es el momento de adentrarnos en el JavaScript. Lo primero que debemos hacer es cargar las imágenes que deseamos mostrar en el Slider. Si nunca has cargado imágenes mediante JavaScript, te sorprenderá lo sencillo que es. En primer lugar, declararé una variable para almacenar las imágenes, y dado que tenemos varias, utilizaremos un array. Para ello, inicializamos el array de imágenes de la siguiente manera: var imagenes = []. Luego, dentro de los corchetes y separadas por comas, especificamos la dirección URL de cada imagen. Es exactamente igual que cuando cargas una imagen directamente en el HTML utilizando el atributo "src". Abres comillas y dentro de ellas colocas la dirección de la imagen. En mi caso, tengo las imágenes guardadas en una carpeta llamada "Imagenes", y cada dirección está separada por una barra diagonal, seguida del nombre de la imagen y, finalmente, un punto y el formato de la imagen. Puedes ver un ejemplo en el código de más abajo.

Visualizando tu Primera Imagen desde JavaScript

Ahora que hemos cargado las imágenes, el siguiente paso es asegurarnos de que, cuando la página se cargue, se muestre automáticamente la primera imagen del array. ¿Cómo lo logramos? Primero, hacemos referencia al objeto Document, seguido del nombre del elemento IMG (imagen) donde deseamos mostrar la imagen. Luego, utilizamos la propiedad SRC para especificar la dirección URL de la imagen que queremos cargar. Usamos el signo igual (=) y finalmente seleccionamos la posición de la imagen que deseamos mostrar dentro del array.

Entiendo que esto puede parecer un poco complicado al principio, así que permíteme explicarlo de otra manera. Mientras observas el código a continuación, permíteme explicarlo en orden inverso. Con esta línea de código, estamos diciendo lo siguiente: "Quiero cargar la imagen que se encuentra en la posición cero del array 'imagenes' (imagenes[0]) y colocarla dentro del atributo SRC, que se encuentra dentro del elemento con el nombre 'Imagen', que a su vez está dentro del objeto Documento (document)". Ahora, con suerte, todo debería estar claro hasta este punto, y podemos continuar más abajo.

Implementando la Funcionalidad del Deslizamiento a la Derecha

Ya hemos cargado las imágenes, y ahora la primera imagen del array se muestra automáticamente. El siguiente paso es comenzar a implementar el cambio de imagen cuando se haga clic en los íconos, empecemos con el de la derecha.

Primero, declararemos una función llamada MoverDerecha, seguida de paréntesis y llaves. Por ahora, déjala así. Hablando de la funcionalidad, necesitamos que cada vez que se haga clic en el ícono de la derecha, se ejecute esta función que acabamos de declarar. Por lo tanto, requerimos un Event Listener que detecte este evento en particular. El primer paso consiste en almacenar el ícono de la derecha en una variable, de esta manera (var SliderDerecho = document.querySelector(".slider-derecho")). Siempre es útil revisar el código que se encuentra más abajo para comprenderlo mejor. Ahora, debajo de la función, indicamos que cada vez que se haga clic en el ícono de la derecha, se ejecute la función MoverDerecha, y esto se logra así: (SliderDerecho.addEventListener("click", MoverDerecha)).

A continuación, encontrarás el código completo hasta este punto para que lo examines y puedas copiarlo. Continuaremos más adelante.

Cambio Dinámico de Imágenes con Cada Interacción

¿Qué necesito hacer dentro de la función? Pues necesito que cada vez que se ejecute la función se muestre una imagen diferente, y para esto, solo necesito un contador que se incrementará en uno cada vez que se ejecute la función. El primer paso es declarar la variable contador con el valor cero (var Contador = 0) en el ámbito global. El segundo paso es entrar en la función y, primero que todo, incrementar en uno la variable Contador (Contador++). El tercer paso es modificar nuevamente el DOM, pero esta vez mostrando la nueva imagen. ¿Cómo lo hago? Pues voy a hacer exactamente lo mismo que hice para mostrar la imagen que ocupa la posición cero en el array. Para hacerlo más sencillo, solo copio y pego esa línea de código dentro de la función. La única diferencia es que, en lugar de pasarle la posición exacta que queremos que se visualice, vamos a pasarle la variable Contador. De este modo, cada vez que se ejecute la función, se visualizará la imagen con el número de posición equivalente al de la variable Contador (la siguiente imagen). Esto se puede observar en el código a continuación.

La Necesidad de Condiciones en la Navegación de Imágenes

Si copias el código en tu editor, podrás notar que todo está perfecto, PERO, llegará el momento en que el contador supere el número de imágenes y, en ese momento, aparecerá un error. Esto es algo lógico, ya que cada vez que se ejecuta la función, el contador se incrementa en uno y no le hemos puesto ningún límite. Esto se soluciona de manera sencilla con una condición que dice: si el contador supera la cantidad de imágenes, el contador debe volver a cero. De este modo, cada vez que llegue a la última imagen, volverá a la primera de manera infinita. Podría quedar de este modo, por ejemplo: if (Contador > 3) { Contador = 0 }.

Tal vez lo sepas y no necesites una explicación, pero por si acaso, podría surgirte una pregunta lógica: ¿Por qué 3, si en realidad son 4 imágenes? La respuesta es que, como en el array, el posicionamiento comienza desde cero (0), significa que la posición de la última imagen siempre será un número menos que la cantidad de imágenes, ya que cero es igual a la imagen número 1. Entonces, 3 es igual a la imagen número 4.

Ahora bien, realmente no está mal comparar el contador con el número 3 (if (Contador > 3)), pero esto podría causar problemas cuando decidas cargar más o menos imágenes en tu slider. Tendrías que revisar el código nuevamente y ajustar ese número según la cantidad de imágenes que hayas cargado. Lo más lógico es hacer la comparación con el número de imágenes actual, de este modo no importará la cantidad de imágenes que decidas agregar en el futuro, el código siempre funcionará. Así quedaría: if (Contador > imagenes.length – 1). ¿Por qué restar 1? Por la razón que se explicó anteriormente (la cantidad de imágenes siempre es 1 más que el número de la última posición de una matriz). A continuación, te dejo el código.

Integrando el Desplazamiento hacia la Izquierda

Si copias el código en tu editor, verás con tus propios ojos que funcionará perfectamente. Cada vez que hagas clic, pasará a la siguiente imagen y, cuando llegue a la última, volverá a la primera sin importar cuántas veces se repita ese bucle. Además, ahora tenemos a nuestro favor que no importa si quieres agregar o eliminar algunas imágenes, el código seguirá funcionando sin errores.

El desplazamiento hacia la derecha está completamente listo. Ahora nos toca hacer lo mismo para el desplazamiento hacia la izquierda. Los primeros pasos son los mismos. Necesitamos agregar un Event Listener al ícono de la izquierda, por lo tanto, el primer paso es almacenarlo en una variable, quedando así: var SliderIzquierdo = document.querySelector(".slider-izquierdo"). Ahora, creamos una función con el nombre MoverIzquierda y automáticamente le agregamos el Event Listener al ícono que acabamos de almacenar en la variable. Pero esta vez, queremos que se ejecute la función MoverIzquierda, por lo que quedaría así: SliderIzquierdo.addEventListener("click", MoverIzquierda).

Hasta este punto, no creo que haya dudas, ya que es lo mismo que hicimos con la función MoverDerecha. Ahora, dentro de la función, en lugar de sumar el contador, lo restamos en uno (Contador–) y luego mostramos la imagen correspondiente (document.Imagen.src = imagenes[Contador]). Hasta aquí todo está bien, pero si depuras el código, te encontrarás con un problema: cuando das clic en el ícono de la izquierda, el valor del Contador se vuelve -1 y esa posición no existe dentro del array. Para solucionarlo, volvemos a entrar en la función y, después de restar el contador, creamos una condición que le indique que si el contador es menor que cero, entonces el contador debe ser igual al número de imágenes menos uno. Quedaría de este modo: if (Contador < 0) { Contador=imagenes.length – 1 }.

A continuación, tienes el código hasta este punto. Échale un vistazo para aclarar cualquier duda y luego seguimos con el siguiente paso.

Automatización del Slider: Movimiento sin Interacción Manual

Hasta aquí todo va de maravilla. Al menos de modo manual, el slider funciona perfectamente, tanto hacia la izquierda como hacia la derecha. Sin embargo, al principio te dije que lo haríamos automático y manual al mismo tiempo, así que continuemos con este paso.

Para hacer que el Slider se mueva automáticamente es super sencillo. Solo necesitas agregar un setInterval que indique que cada dos segundos (por ejemplo), se ejecute la función MoverDerecha. La lógica humana siempre interpreta que ir a la derecha es avanzar, ¿verdad? 😂, así que quedaría de este modo: setInterval(MoverDerecha, 2000). Esta línea de código se agrega justo debajo de la función MoverDerecha.

Sin embargo, al hacerlo de esta manera, nos encontramos con un inconveniente. Imagina este escenario: el usuario decide regresar para ver la imagen anterior porque le llamó la atención. Si el usuario hace clic en el ícono de la izquierda a los 1.5 segundos, significa que solo tendrá medio segundo para visualizar la imagen antes de que el slider se mueva automáticamente a la siguiente. Hemos puesto el ejemplo con la imagen anterior, pero lo mismo sucederá si decide ver la siguiente. Esto es fácil de solucionar; simplemente, cada vez que el usuario decida hacer clic en cualquiera de los íconos, reiniciamos el intervalo. Dejemos de hablar y vayamos a la acción, como dijo MiduDev en un podcast: "Hablar es FÁCIL, muéstrame el CÓDIGO" 😂.

Reiniciando el Intervalo: ¿Por qué y Cómo?

Realmente, la parte más complicada ya está hecha; ahora viene la parte más fácil. Lo primero que necesitas saber es que para limpiar el intervalo, utilizamos el método clearInterval(). Dentro de los paréntesis, le pasamos el intervalo que necesitamos limpiar. Entonces, lo que vamos a hacer es almacenar este intervalo (setInterval(MoverDerecha, 2000)) dentro de una variable para poder limpiarlo después. Quedaría de este modo: var Intervalo = setInterval(MoverDerecha, 2000). Ahora tenemos listo el intervalo para ser reiniciado cuando sea necesario. Como mencioné antes, vamos a limpiarlo cada vez que se haga clic en uno de los íconos.

Ahora nos dirigimos al primer evento, que es el del ícono derecho. En lugar de decirle que ejecute la función MoverDerecha al hacer clic, vamos a decirle que ejecute una función anónima: SliderDerecho.addEventListener("click", function(){}). Luego, dentro de esta función, realizamos los pasos necesarios. El primer paso es limpiar el intervalo con el método clearInterval, de este modo: clearInterval(Intervalo). Luego de limpiar el intervalo, ejecutamos la función MoverDerecha. Por último, una vez que se haya ejecutado esa función, volvemos a lanzar el intervalo. ¿Cómo? Solo necesitamos reasignar la variable de este modo: Intervalo = setInterval(MoverDerecha, 2000).

Todo está listo para el ícono de la derecha, pero falta hacer lo mismo con el de la izquierda. Como es exactamente lo mismo, no creo que sea necesario explicarlo; puedes verlo por ti mismo en el código a continuación.

Este modelo es realmente genial y bastante fácil de crear, pero hay algo que no me gusta, y es que no podemos agregarle una transición en el cambio de imagen. Puedes intentarlo tú mismo; ve al CSS y agrega una transición en el contenedor de las imágenes o incluso en la propia imagen, y verás que no funciona. Simplemente no hay una transición entre mostrar y ocultar. Pero, como este es un lenguaje y solo es cuestión de pensar un poco, siempre hay una manera de hacer las cosas. Si le das una opacidad inicial de cero a la imagen, luego declaras una clase llamada ".visible" con una opacidad de 1, y finalmente dentro de las funciones (MoverDerecha y MoverIzquierda), agregas dos temporizadores setTimeout. Uno para agregar la clase ".visible" unos milisegundos después de que aparezca la imagen y otro para quitarla unos milisegundos antes de cambiar la imagen. Con eso, obtienes la sensación de transición entre una imagen y otra.
